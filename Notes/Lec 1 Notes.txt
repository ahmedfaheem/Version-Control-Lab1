‎git reset --hard <sha>     -- return to this commit and delete the current and no modifications in working dirctory and stages
 ‎git reset --soft <sha>     -- return to this commit but modification at staged
 ‎git restore --staged <file>    -> return file to unstage  in working direcotry
 ‎git restore <file>  -->  Deletes changes from the file and restores it to the last commit of it  -- like undo
git reflog --> get last history  to restore deleted commit -- saved for 90day

git commit --amend  -m "name"  --> changes the last commit’s message (or content/author), but Git does not edit the commit directly — it creates a new commit to replace it.
The old commit is effectively deleted and replaced, so the new commit gets a new SHA so you can return to last also

git reset --hard HEAD^^    back to two commit
git reset --hard HEAD~2     back to two commit
